useEffect(function,Array)

  • 类似于类组件的生命周期，但又不能将他理解为函数式组件的生命周期
  • 接收两个参数，function是一个副作用函数，Array中放入的是副作用函数依赖的变量，当Array是一个空数组时，表示不依赖任何变量，副作用函数只执行一次。当传入这个依赖变量并且这个变量发生改变时会重新执行一遍副作用函数。
  • 第二个参数有点类似于componentDidMount 和 componentWillUnmount 的思维模式，并且 React 会等待浏览器完成画面渲染之后才会延迟调用 useEffect，因此会使得处理额外操作很方便。
  • useEffect可以多次调用创建多个副作用函数。使用useEffect就可以模拟出类组件从创建到更新到销毁的生命周期。但是这个狗子函数并不是生命周期，只是基于依赖的副作用函数。

useCallback(function,Array)

  • 如果说useState是为了让函数组件记住状态，则useCallback则是让组件记住在内部注册过的函数。因为初始化组件后，注册事件函数，后续的更新操作中也会重新定义注册函数。
  • 接受两个参数，第一个为注册创建的函数，第二个为组件因为更新需要重新注册创建本函数所需要的依赖。
  • useCallback可以防止因为组件重新渲染，导致方法被重新创建 ，起到缓存作用; 只有第二个参数 变化了，才重新声明一次。

useMemo(() => fn, [inputs])

  • useCallback的功能完全可以由useMemo代替，唯一的区别是useCallback 不会执行第一个参数函数，而是将它返回给你，而useMemo会执行第一个函数并
    且将函数执行结果返回给你。也可以理解为计算属性。

useRef(保存引用值)

  • useRef 返回一个可变的 ref 对象，其 .current 属性被初始化为传入的参数（initialValue）。返回的 ref 对象在组件的整个生命周期内保持不变。
  • 常用有于两种情况：1、获取组件内的dom节点，或者子组件的组件对象；2、保存一个变量————除了使用useSate保存状态之外，还可以用useRef，在后续的组件更新中，变量的值不会丢。

自定义hooks

  • 自定义hooks说白了就是把重复的代码拿来封装复用，但是需要注意的是，React自定义 Hook 是一个函数，其名称以 “use” 开头，函数内部可以调用其他的 Hook。
    • 自定义 Hook 必须以 “use” 开头吗？必须如此。这个约定非常重要。不遵循的话，由于无法判断某个函数是否包含对其内部 Hook 的调用，React 将无法自动检查你的 Hook 是否违反了 Hook 的规则。
    • 在两个组件中使用相同的 Hook 会共享 state 吗？不会。自定义 Hook 是一种重用状态逻辑的机制(例如设置为订阅并存储当前值)，所以每次使用自定义 Hook 时，其中的所有 state 和副作用都是完全隔离的。
