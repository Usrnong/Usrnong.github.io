react-router-dom@5，前端路由设置

发表于2022-07-18|更新于2023-07-14|react教程

react路由小结 react-router-dom@5

  • 1、react的路由是使用react-router-dom插件实现，这是一个针对React而设计的路由解决方案、可以友好的解决React components 到URl之间的同步映射关系。

  • 2、路由安装，常用的路由模块包括有 BrowserRouter 、HashRouter 、Switch、 Route、 NavLink、Redirect 模块。

  • 3、各个路由代码一般由 BrowserRouter 或者 HashRouter 作为标签包裹在内，同级路由代码由Switch 作为标签包裹在内，每一个路由写在Route标签内，包含属性path和component或者rander 以及可选属性 exact 精准匹配。

  • Redirect 路由重定向。包含属性 from（重定向的规则） 、 to（重定向到某路径） 、exact（精准匹配）；

  • NavLink 声明式导航：to属性接导航路径;activeClassName=”active”属性设置动态样式；

  • 嵌套路由：在某些组件或页面中继续使用 Switch 包裹子级路由Route标签；

  • 编程式导航：类组件使用this.history.push 方法；函数组件使用 useHistory 狗子的方法返回的值再进行push；

  • 路由传参的方式：
    1、动态路由传参 ：需在设置路由路径时就设置好需要传入的参数 <Route path='/detail/:id' component={Detail}></Route> 。子页面通过传入的属性获取 props.match.params.id
    2、query传参：不用在设置路由时就设置好参数，只要在编程式导航时将参数写入：history_.push({pathname: '/detail',query: { id: num }}) ，子页面通过props.location.query.id获取
    3、通过state传参：与query类似 this.props.history.push({ pathname:'/user',state:{day : 'Friday' } }) 。子页面通过传入的属性获取 this.props.location.state.day

    • query和state传参在页面重新刷新之后就不存在了。

  • 路由拦截：通过Router的rander属性，自定义路由拦截的规则
