常见前端面试八股文（02）

发表于2023-07-30|更新于2023-10-18|前端面试题

1、css水平居中的方式有哪些？

  • 1、margin-left ===》absolute fixd 然后定位50%
  • 2、margin-auto ===》absolute fixd 偏移为0，最后margin-auto
  • 3、flex

2、标准盒模型和怪异盒模型？

  • 区别：

    box-sizing: content-box (标准计算) border-box（怪异）

  • 标准计算：margin+border+pindding+content
  • 怪异计算: content+margin

3、什么是重绘和重排？

  • 重绘——重新绘制，改变外观，布局无改变；
  • 重排——重新排列，dom改变、style改变

4、什么是防抖和节流？

  • 防抖和节流都是为了防止高频率的js代码执行；
  • 防抖——最后的操作为标准，等待某种行为停止后，加以时间间隔进行操作；
  • 节流——等待某种时间间隔后进行操作，事件在规定事件内只能触发一次。

5、felx属性？

  • 针对子元素的空间排列；包括：
    • 1).大小、空间分布
    • 2).流动方向
    • 3).两轴的对齐与分布
    • 4).顺序
  • 主轴排布：just-content：flex-start …..
  • 主轴纵横：flex-direction：row ….
  • 辅轴对齐：align-item：conter ….
  • 个别对齐：align-self：

6、vue3中 ref和reactive有什么区别？

  • 都是为了在模板中使用的数据能够达到双向绑定的目的。定义方法一致，都是调用这个函数并且将需要的双向绑定的数据传入。不同的是使用reactive具有一定的局限性：
    • 有限的值类型：它只能用于对象类型 (对象、数组和如 Map、Set 这样的集合类型)。它不能持有如 string、number 或 boolean 这样的原始类型。
    • 不能替换整个对象：由于 Vue 的响应式跟踪是通过属性访问实现的，因此我们必须始终保持对响应式对象的相同引用。这意味着我们不能轻易地“替换”响应式对象，因为这样的话与第一个引用的响应性连接将丢失
    • 对解构操作不友好：当我们将响应式对象的原始类型属性解构为本地变量时，或者将该属性传递给函数时，我们将丢失响应性连接
    • 由于这些限制，我们建议使用 ref() 作为声明响应式状态的主要 API。

7、veu的响应式原理？

  • vue2:其核心原理是通过Object.defineProperty把这些 需要成为响应式的数据也就是data 全部转为 getter/setter形式。当你把一个普通的 JavaScript 对象传入 Vue 实例作为 data 选项，Vue 将遍历此对象所有的 property 将其变为这种形式。当模板使用到这些数据时，也就是render渲染节点时，每一个使用的数据就会形成一种依赖关系收集起来，而每个组件实例都对应一个 watcher 实例，它会在组件渲染的过程中把“接触”过的数据 property 记录为依赖。之后当依赖项的 setter 触发时，会通知 watcher，从而使它关联的组件重新渲染。由于 JavaScript 的限制，Vue 不能检测数组和对象的变化

  • vue3:其核心原理是proxy对象代理原始数据，即将数据包装成具有proxy对象代理的形式，传入的原始数据在经过代理后达到响应式数据的目的。其中ref传入普通的基本数据类型时保留getter / setter的形式，这也是为什么在使用响应式数据时需要采用.value形式的原因。当传入的数据是对象时，才会采用proxy的形式进行数据代理。
    在vue3中，一个数据被定义成为响应式数据对象，数据变化的过程可称之为effect，收集数据的变化可称为track，触发变化的称为trigger，在收集变化track是会将所有的effect将会存到一个set集合里。数据和数据变化会在全局形成一个 WeakMap<target, Map<key, Set<effect>>> 的映射表。在 trigger() 之中会查找对应属性的对应effect并且全部执行它们。

8、vue2和vue3有什么区别？

  • 1、响应式原理不同
  • 2、vue3 采用了选项式api的形式，源码改成了函数式编程，方便按需引入，而tree-shaking必须配合按需引入的写法才能实现，所以vue3的打包体积会相对于vue2更小
  • 3、生命周期改变——取消了beforeCreate created，此阶段用setup代替
  • 4、v-if优先级高于v-for
  • 5、template可以不有且只有一个根节点
  • 6、性能优化更好——则加了静态节点标记，dom更新时不对静态节点进行比对，从而增加效率
  • 7、vue3不推荐使用mixin进行复用逻辑提取，而是推荐改用hook
  • 8、v-model应用于自定义组件时，监听的事件和传递的值有所改变——vue2传递的时value值监听的是change/input事件；vue3传递的是modelValue监听的是update:modelValue
  • 9、更好的TS配合
  • 10、css新增：
    • 深度选择器 :deep
    • 插槽选择器 :slotted(div)
    • 全局选择器 :global(.red)
    • 支持CSS Modules： 默认以$style 对象暴露给组件 <p :class="$style.red"> ;
      可以自定义注入module名称 <p :class="classes.red">red</p> <style module="classes">.red {color: red;}</style>
    • 在style中可以使用script的变量等

9、vue3中的watch和watchEffect有什么区别？

  • watch要显示明确监视哪些数据，显示指明监视回调。具有惰性，第一次运行时不执行，只有设置immediate才会执行，监视ref响应式数据时可以获得初始的值；
  • watcheffect自动收集依赖数据，依赖更新会自动执行本身；不需要指定监听数据，首次挂载就会执行，无法获取原始的数据值，只能得到变化后的值

    watch函数有两个小坑：
    监视reactive定义的响应式数据（该数据为一个对象，因为reactive只能定义数组或对象类型的响应式）时：oldValue无法正确获取，会强制开启深度监视，deep配置不生效。
    监视reactive定义的响应式数据中的某个属性时，且该属性是一个对象，那么此时deep配置生效。

10、项目优化问题：

Ⅰ 内存泄漏

  • 全局存有大量数据未释放——大量的业务逻辑、循环等、dom元素移除但还作为js的引用时(已分离dom)
  • 闭包场景使用完毕没有释放内存空间
  • console未移除，内部对其产生强引用
  • 大量的事件监听未移除

    结合浏览器开发者工具（F12）进行内存快照分析，找出相应代码块进行释放

Ⅱ vue首次加载白屏

  • 项目组件大，首次加载组件过多导致
    • 路由懒加载
    • 异步组件
    • 定制首页内容
    • 使用Suspense提升体验
  • 动画加载导致
    • 减少首屏动画，定制首屏简单的动画实现等待，过渡到复杂动画

Ⅲ 开发环境优化(以webpack为例)

  • 使用热更新功能。开启hmr，如果是4以下版本，需要在代码判断module.hot。css不必处理，内部style-loader实现了热更新
  • 开启devtool:”source-map”，使代码调试更加友好 （eval-source-map）

Ⅳ 生产环境优化

  • oneOf:优化module中loader的执行方式 ，如果你的module中配置了多个loader，webpack打包每一个文件的时候都会有序执行每一个loader配置，相当于每一个文件都会被所有的loader过一遍，会减缓webpack的打包速度
  • 生产环境的缓存配置：babel是对js文件进行处理编译处理、兼容性处理，编译成浏览器能够识别的语法。假设js有很多个模块，若开发过程中只改变其中一个模块，编译时会重新打包所有模块，降低webpack的打包速度，跟HMR功能类似，只不过HMR是浏览器刷新时的重新加载的模块，这个指的是webpack打包时的模块.开启babel缓存只需要在babel-loader下的options下面添加：cacheDirectory: true
  • 文件资源缓存：基本思路：每个文件附带版本号，每次请求若版本号变化时，会重新获取资源而不是从缓存中获取。每一次webpack打包都会产生一个hash值，将此hash值作为输出文件的名称的拼接可以有效解决这个问题。babel缓存让第二次打包构建的速度更快，文件资源缓存让上线运行环境的缓存更好使用。
  • tree shaking：去除无用代码，忽略一些无用文件（css\js）减小构建后文件的体积。自动开启
  • code split 代码分割：将体积较大的文件分成多个文件
    • 多入口文件配置
    • 使用optimization配置，splitChunks：{chunks: “all”} ，将多个文件引入相同的包单独打成一个文件。
    • 使用js代码完成单独打包：import(/* webpackChunkName: ‘logon’ */‘./logon’).then(({logon})=>{ ….. 最常用的方式
  • 使用externals——忽略某些包，不进行打包。然后通过cnd加速引入
  • 多线程打包—— thread-loader：webpack单线程处理程序，如果打包的文件很多，工程量比较大，则打包速度就会降低，所以此时可以考虑多线程打包。多进程打包是需要开启时间的，启动时间大约为600ms，进程通信为也需要时间，只有打包时间需要较长的打包才会使用多进程打包。

Ⅴ 其他优化：

  • 减少http请求
  • 使用雪碧图
  • 合理使用浏览器Storage
  • 图片懒加载
