常见前端面试八股文（01）

发表于2023-06-30|更新于2023-08-27|前端面试题

1、客户端与服务端通常通过什么机制实现登录验证？

  • HTTP Basic —— 每个接口都接受账号和密码
  • OAuth —— 开放授权。允许第三方应用访问该用户在某一服务器上访问，无需将用户名和密码提供给第三方。每一个授权令牌接受一个特定的第三方系统，允许在特定时段访问（QQ、微信），适用于个人消费类的互联网产品，如社交app。
  • cookie Auth
  • Token Auth —— 无状态，不需要存储session信息。去耦，不需要绑定特定的身份认证方案。

2、什么是响应式以及css实现响应式的机制？

  • 根据不同的浏览器视口将页面（dom）以不同的形式展示出来的web应用程序
    1. 媒体查询
    2. flex
    3. grid

3、html5的媒体标签：

  • audio video margaee

4、如何理解dom与bom？

  1. bom（浏览器对象模型）——是javascript开发web应用程序的核心，提供了与网页无关的浏览器功能对象Window，window可作为js的global对象，通过var声明的全局变量和函数都成为window的属性和方法。
  2. dom（文档对象模型）——是指HTML和XML文档编程接口。是多层节点构成文档、开发者通过它添加和删除、修改文档中的节点。

5、jq中的 get（） 和eq（）有什么区别？

  • get（）用与转换成dom对象， eq（）是用于选择节点指定的index元素的方法

6、对于js模块化的理解？

  • commonjs 是一种广泛使用的javascript模块化规范，核心思想是通过require方法来同步加载依赖的其他模块，通过modulem.exports到处需要暴露的接口。commonjs的流行得益于nodejs也采用了这种方式，后来这种方法也被应用到网页开发中。

    优点：代码可复用在nodejs环境，很多第三方采用的模块化方式。

    缺点：无法直接在浏览器环境下运行，必须通过工具转换成es才能运行。

  • AMD 也是一种模块化规范，与commonjs不同的是它采用异步的方式去加载模块。AMD规范是为了解决针对浏览器环境的模块化问题。最具有代表性的就是requirejs。

    优点：直接在浏览器环境下运行，也可在node环境下运行，可异步加载，可并行加载。

    缺点：JavaScript原生环境中没有支持AMD，需要导入实现了AMD的库后才能正常使用。

  • ES6 是ECMA提出的一种新的模块化规范，它在语言层面上实现了模块化，浏览器和node都宣布要原生支持该规范，它将逐步取代commonjs和AMD，成为最终的解决方案。

    缺点：无法直接运行在大部分的JavaScript环境，必须通过工具转换成为标准es5后才能运行。

7、什么是闭包？

  • 闭包是一个函数加到创建函数的作用域的链接，闭包关闭了函数的自由变量。

    内部函数可以访问到外部函数的变量；变量会驻留在内存当中，会造成内存损耗的问题（把闭包函数清除即可）

8、js延迟加载的方式？

  • asyn：与html解析一起，并不是顺次执行js，谁先加载执行谁
  • defer：等html全部加载才会执行js，并且顺次执行js。

9、null和undefined有什么区别？

  • 最初设计时借鉴了java只设计了null作为“无”的值，但是null是一个对象。会被转换为0。很不容易发先错误。而undefined的出现是为了填补之前的不足。null表示为“无”的空对象指针，转换值为0。undefined表示“无” 的原始值，转换为值NaN.

10、js的宏任务和微任务？

  • js是单线程的。js的执行流程为：

    执行同步===》事件循环【宏（定时器）、微任务（promise then）】

要执行宏任务的前提就是清空了微任务。

同步=》微任务=》宏任务=》微…..（循环）

11、理解js的作用域？

  • 除了函数外，js是没有块级作用域的；
  • 作用域链：内部可以访问外部变量，但是外部不能访问内部变量（如果内部存在，先查找内部，否则向外部查找）
  • 注意声明变量是用var还是其他的关键词（window）
  • js 有变量提升机制（变量悬挂声明）
  • 面试规则：

    1.本层作用域有没有变量（变量提升）；
    2.注意js除了函数外，有没有块级作用域；
    3.优先级：声明变量>声明普通函数>变量提升
    4.普通函数声明是不看写函数的顺序的

12、js对象考题？

  • 所有的对象都是通过new出来的，所以它们不相等！
  • 对象注意引用类型；
  • 对象的key都是字符串类型的；
  • 查找对象的方法或属性顺序：

    现在对象本身找—->构造函数—->对象原型—->构造函数原型—->对象上层原型

13、js判断是不是数组？

  • 1、isArray()
  • 2、instanceof arr
  • 3、通过原型 prototype.toString.call(arr).indexOf("Array") > -1
  • 4、Array.prototype.isProtypeOf(arr)
  • 5、arr.constructor.toString(),indexOf('Array') > -1

14、js数组去重？

  • 1、 new Set ==》 Array.from(new set(arr)) 或者 [...new Set(arr)]
  • 2、indexOf 循环
  • 3、sort 先排序，然后对比和前一项是否相等；

15、js的new操作符做了什么？

  • 1、创建空的对象；
  • 2、将空对象的原型指向构造函数的原型；
  • 3、将空对象作为构造函数的上下文（改变this指向）；
  • 4、对构造函数有返回值做判断处理。

16、js有哪些继承方式？

  • 1、es6 的 extends
  • 2、原型链继承——子类的prototype等于父类的实例；
  • 3、构造函数继承——Parent.call(this);
  • 4、组合继承

17、js中的call、apply、bind有什么区别？

  • 共同点 ：功能一致，改变函数体的this指向。
  • call、apply会立即执行，bind返回函数体本身，需要加（）才能执行；
  • 参数不同，apply第二个参数是数组，call和bind挨个写。
